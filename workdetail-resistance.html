惡靈古堡【反抗】

非對稱線上多人對戰遊戲

UI / UX Designer
(影片為官方宣傳影片，非本人作品)

惡靈古堡：反抗是一款一對四非對稱對抗遊戲，四位生存者必須一同合作闖關，抵抗首腦不下的層層陷阱，通關獲得勝利。四位生存者各有補師、坦克、駭客與近戰等天賦，在關卡中透過各自的能力協助彼此，抵抗來自首腦的攻擊。而首腦則運用第一人稱的上帝視角，在關卡中設置各式各樣的機關與殭屍、附身到魔王殭屍，阻擋生存者在時間內完成通關。

(實際遊玩影片範例)

我的任務

組織MainFlow的資訊架構，協助規劃MainFlow與HUD的流程與介面，並製作高、底保真原型與規格文件。設計出遵從惡靈古堡本系列作品的介面與操作特性，再考慮反抗核心玩法與機制的介面與操作模式。

負責項目

Main Flow 的介面與規格，EIS(環境互動系統)介面與動態設計，玩家編輯系統規劃與設計。撰寫設計文件與日方以及UI Artist進行協作與交接。

(時間0:13至1:00為作品集範例)

Main Flow 介面

專案的第一個任務為，規劃Mainflow的資訊架構，整合系統與遊戲需求，建立SiteMap，並設計Mainflow的框線稿，確立每個頁面該有的資訊以及功能，並整合成合適的介面版型，最後串接成可測試的UI Flow。同時，隨著開發階段，慢慢迭代Mainflow的架構與介面，並隨時調整以及修改設計規格文件。

玩家預設編輯系統

惡靈古堡：反抗提供玩家預設編輯系統，讓首腦能設計各式陷阱與環境配置，減輕首腦在關卡開始時面對四位生存者玩家的壓力。我負責設計編輯系統的版面架構、環境裝置的編輯方式。與企劃以及開發討論物品擺設的精準程度、操作方式與可行性。

(時間1:21至3:14)

HUD，發散與收斂

HUD對於玩家來說至關重要，其所呈現的資訊與何時呈現，將影響玩家在遊戲中理解自身、隊友與勝負狀況，進而影響整體的遊戲體驗。反抗在遊戲中所呈現的資訊非常的大量，因此為了妥善規劃HUD介面，我們運用發散思維，將HUD上會可能需要呈現的資訊給羅列出來，運用關聯性歸納並分類，再運用權重收斂，依據資訊的重要性分配到介面的位置上，以及使用時機分配到動態呈現的資訊上。

EIS環境互動系統設計

反抗的一大特色為首腦的遊玩方式，透過監視器的畫面，在遊戲場景中設計陷阱與殭屍，阻撓生存者們前往下一個關卡，因此環境互動系統將會是首腦遊玩的主要方式。在EIS中我負責設計操控場景物件的互動方式，整理規格，並運用AE做出原型與UI Artist以及開發確認具體呈現樣貌。
